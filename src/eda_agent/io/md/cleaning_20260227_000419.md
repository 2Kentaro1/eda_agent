## 🔄 DATA_CLEANING_AGENT: クリーニングタスクの開始

EDAの結果に基づき、以下のデータ品質に関する課題が特定されました。これらを解決するため、DATA_CLEANING_AGENTとしてデータクリーニングを実行します。

### クリーニングが必要な項目:
1.  **`datetime`カラムのデータ型変換**: 現在は既に`datetime`型ですが、初期段階での確認と、必要に応じての変換が重要です。
2.  **`precipitation`カラムの処理**: `--`という特殊な文字列が含まれており、これを数値として扱えるように`NaN`に変換し、データ型を`float`にする必要があります。
3.  **欠損値の処理**:
    *   `kcal`：数値データであり、欠損値が存在します。
    *   `remarks`, `event`：カテゴリカルデータであり、欠損値が存在します。
    *   `payday`：バイナリデータ（1またはNaN）であり、欠損値が存在します。

### クリーニングステップ

import numpy as np

# DataFrameを再構築 (EDAで生成した特徴量も含む)

# ここでは説明のために再定義します。
